What Is a UUID v4 API Key?

A UUID (Universally Unique Identifier) v4 is a 128-bit random identifier formatted as xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx — 32 hex digits arranged in 5 groups separated by hyphens. The 4 in the third group indicates version 4 (random), and the y position encodes the variant bits.

UUID v4 is the standard format for API keys because it provides 122 bits of randomness (enough for collision resistance at any realistic scale), is universally recognisable, and works as an opaque token that reveals nothing about its issuer or structure.

Only use CSPRNG-generated UUIDs for API keys. Math.random()-based UUID generators are predictable. This tool uses crypto.getRandomValues() — the same entropy source as your OS.

How to Use an API Key in Your App

1
Copy a UUID above and store it as an environment variable: API_KEY=xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx. Never hardcode it in source code.
2
Pass it in the Authorization header on API requests: Authorization: Bearer <key> or as a custom X-API-Key: <key> header.
3
Store it hashed in your database (SHA-256 is fine for lookup; bcrypt for higher sensitivity). Only show the raw key to the user once at creation.
4
Provide key rotation — let users regenerate a key without losing access to their account.
// Node.js — verify incoming API key
import crypto from 'crypto';

function verifyApiKey(incoming, storedHash) {
  const hash = crypto.createHash('sha256').update(incoming).digest('hex');
  return crypto.timingSafeEqual(Buffer.from(hash), Buffer.from(storedHash));
}
