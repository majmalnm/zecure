Why Learn Linux Commands?

Linux powers over 96% of the world's top 1 million web servers, all major cloud platforms, and virtually every container runtime. Whether you're a developer, sysadmin, or security engineer, fluency in the terminal is one of the highest-leverage skills you can build.

The commands in this cheat sheet were chosen for real-world usefulness — not textbook completeness. Each one earns its place by solving common problems faster than any GUI alternative.

File System Commands

File system commands are the foundation. ls -la reveals hidden files and permissions at a glance. find is the power tool for locating files by name, type, size, modification time, or permissions — including SUID binaries. rsync beats cp for large transfers because it's resumable, differential, and works over SSH.

Tip: Combine find with -exec or pipe to xargs to act on results. Example: find . -name "*.log" -mtime +30 | xargs rm -f deletes logs older than 30 days.

Text Processing: grep, sed, awk

These three commands handle most log analysis and data transformation needs on the command line.

  • grep — filters lines matching a pattern. Add -r to search recursively, -n for line numbers, -i for case-insensitive, -v to invert (exclude matches).
  • sed — stream editor for in-place substitution. sed -i 's/old/new/g' file replaces every occurrence. The -i flag edits the file in place.
  • awk — a mini programming language for structured text. awk '{print $2}' extracts the second field from each line. Combine with BEGIN and END blocks for aggregations.

Chain them together: grep "ERROR" app.log | awk '{print $4}' | sort | uniq -c | sort -rn gives you a ranked count of error sources in seconds.

Networking Commands

ss -tulpn is the modern replacement for netstat and shows all listening ports with associated process names in one shot. Use curl -I to inspect HTTP headers without downloading the body, and nc -zv host port to quickly test connectivity to any TCP port.

For DNS troubleshooting, dig +short gives clean output. Add record type flags: dig MX domain.com +short for mail records, dig TXT domain.com +short for SPF/DKIM.

Security-Relevant Commands

Several commands here are indispensable for hardening and auditing Linux systems:

  • find / -perm -4000 2>/dev/null — lists all SUID binaries, a key step in privilege escalation audits.
  • last and lastb — show successful and failed logins, respectively. Essential for incident response.
  • openssl rand -hex 32 — generates cryptographically secure random strings for secrets and API keys.
  • ssh-keygen -t ed25519 — creates a modern, compact Ed25519 key pair (prefer over older RSA-2048).

Process Management

Know the difference between kill -15 (SIGTERM — ask politely to stop) and kill -9 (SIGKILL — stop immediately, no cleanup). Always try SIGTERM first. Use lsof -i :PORT to find what's occupying a port before killing it blindly. nohup command & keeps a process running after you log out — redirect output with > output.log 2>&1 to capture stdout and stderr.

Frequently Asked Questions

There is no single answer, but grep, find, curl, and ps are among the most universally useful. grep lets you search through files and output, find locates files by name or attribute, curl makes HTTP requests from the terminal, and ps aux shows every running process. Mastering these four covers the vast majority of daily terminal work.

Use find . -name "filename" to search recursively from the current directory. Use -iname for case-insensitive matching. Add -type f to match only files or -type d for directories. Wildcards work: find /var -name "*.log" finds all .log files under /var. For fast indexed searches, install and use locate filename (requires running updatedb first).

Run ss -tulpn | grep :PORT or lsof -i :PORT. Both show the process name and PID occupying a specific port. For example, ss -tulpn | grep :80 shows what is listening on port 80. You may need sudo to see processes owned by other users.

First find the PID with pgrep -l processname or ps aux | grep processname. Then run kill PID (sends SIGTERM — graceful shutdown) or kill -9 PID (sends SIGKILL — forced). To kill by name directly: pkill processname. Always prefer SIGTERM first and give the process a few seconds to clean up before using SIGKILL.

grep -r pattern dir searches recursively through all files in a directory and its subdirectories for lines matching the pattern. Add -l to list only matching filenames, -n to show line numbers, -i for case-insensitive matching, -v to show non-matching lines, or --include="*.py" to limit to specific file types.

ss (socket statistics) is the modern replacement for netstat — it is faster, maintained, and available on all current Linux distributions. netstat is deprecated on many systems. Use ss -tulpn instead of netstat -tuln for the same information. The flags t (TCP), u (UDP), l (listening), p (process), n (numeric) are nearly identical.
