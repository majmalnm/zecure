How to Create a Strong Random Password — A Complete Guide

A strong password is your first line of defence against unauthorised access. This guide explains what makes a password truly secure, how attackers crack them, and exactly what you need to do to stay safe.

Why password strength matters

Every account you have — email, banking, social media, work tools — is protected by a password. If any of those passwords is weak, an attacker doesn't need sophisticated hacking skills. They can simply guess it, or run an automated script that tries the most common passwords in seconds.

The Have I Been Pwned database contains over 600 million leaked passwords. The most common ones — 123456, password, qwerty — can be cracked in milliseconds. A strong random password is the simplest, most effective step you can take toward securing your digital life.

If you use the same password across multiple sites, a leak at any one of them exposes all of them. Use a unique, strong password for every account.

Length is the most important factor

When it comes to password strength, length beats complexity every time. This is because brute-force attacks work by trying every possible combination of characters. Each additional character multiplies the number of possible combinations by the size of the character pool.

Consider the difference:

  • 8 characters, alphanumeric (a-z, A-Z, 0-9, 62 possible values per position): 628 ≈ 218 trillion combinations. A modern GPU can crack this in days.
  • 16 characters, alphanumeric: 6216 ≈ 48 nonillion combinations. This would take centuries at a billion guesses per second.
  • 20 characters, lowercase only (26 values): 2620 ≈ 20 octodecillion combinations. Still stronger than many 8-character passwords using all four character classes.

The takeaway: aim for a minimum of 16 characters, and aim higher for high-value accounts like your password manager master password or email.

Use all four character classes

Once you've prioritised length, the next lever is character diversity. The full printable ASCII character set gives you 94 possible characters per position. Using only lowercase letters gives you just 26 — a much smaller search space.

A strong password generator like Zecure's free password generator lets you toggle four character classes:

  • Uppercase (A–Z) — adds 26 characters
  • Lowercase (a–z) — adds 26 characters
  • Numbers (0–9) — adds 10 characters
  • Symbols (!@#$%^&*...) — adds 32 characters

Use all four. It expands your pool from 26 to 94 characters per position, making each character meaningfully harder to guess.

The common advice to "replace O with 0 or E with 3" is outdated and ineffective. Attackers know these substitutions. Use real randomness instead of predictable transformations.

Understanding password entropy

Entropy is measured in bits and represents the randomness or unpredictability of a password. A password with n bits of entropy would require at most 2n guesses to crack in a brute-force attack.

For a truly random password using all four character classes, entropy works out to approximately 6.55 bits per character. That means:

  • A 12-character password ≈ 78 bits of entropy
  • A 16-character password ≈ 105 bits of entropy
  • A 20-character password ≈ 131 bits of entropy

For comparison, the Password Entropy Explained guide goes deeper into how these numbers are calculated and what they mean in practice.

What to avoid

Knowing what makes a password weak is as important as knowing what makes it strong:

  • Personal information — birthdays, pet names, children's names. This information is often publicly available or guessable from your social media profiles.
  • Dictionary words — single words, even with number substitutions, are vulnerable to dictionary attacks that try every word in a language.
  • Keyboard patterns — qwerty, asdfgh, 12345678 are among the first combinations any attacker will try.
  • Reused passwords — if one site leaks, all sites using the same password are compromised.
  • Short passwords — anything under 12 characters can be cracked by consumer hardware in a reasonable timeframe.

The best approach: use a password manager

The most practical security setup is a password manager that generates and stores unique, strong random passwords for every account. Quick checklist for a strong password

  • Minimum 16 characters (20+ for high-value accounts)
  • Uses uppercase, lowercase, numbers, and symbols
  • Generated randomly — not created from personal information
  • Unique to this account — not reused anywhere else
  • Stored in a password manager, not in a browser or text file

If you follow these five rules, your accounts will be orders of magnitude more secure than the vast majority of internet users. The attackers will move on to easier targets.
