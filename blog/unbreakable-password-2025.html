Why Most "Strong" Passwords Aren't

The classic advice — use a capital letter, a number, and a symbol — dates from a 2003 NIST paper the author later publicly regretted. It produces passwords like P@ssw0rd! that are easy to remember but trivial for modern cracking tools to guess because the substitutions are all predictable.

Modern GPUs can test billions of candidate passwords per second. Dictionaries now include every word in every language plus all their common letter-to-symbol substitutions. A password that "feels" clever is usually one of the first 100,000 guesses.

If you've ever used a password that includes your name, a pet's name, a birth year, or a word with an @ for the letter a — it's almost certainly in a modern cracking wordlist already.

What Actually Makes a Password Strong

Security researchers now agree on three properties that matter:

1. Length is everything

The single highest-impact improvement you can make is adding more characters. Every additional character multiplies the cracking time exponentially. Here's what that looks like for a fully random password using a 94-character set (letters, digits, symbols):

2. True randomness, not human randomness

Humans are terrible at picking random passwords. We gravitate toward familiar patterns, avoid hard-to-type characters, and unconsciously make choices that reduce entropy. The only way to get truly random credentials is to let a cryptographically secure random number generator (CSPRNG) do it for you.

All modern browsers expose window.crypto.getRandomValues() — this is what Zecure uses. It draws from the operating system's entropy pool and is the same source used for TLS key generation.

// How Zecure generates unbiased random passwords
function secureInt(max) {
  const limit = Math.floor(0xFFFFFFFF / max) * max;
  let n;
  do {
    n = new Uint32Array(1);
    crypto.getRandomValues(n);
  } while (n[0] >= limit); // rejection sampling — no modulo bias
  return n[0] % max;
}

3. Uniqueness per site

A strong password used on two sites is only as secure as the weaker site. When that site gets breached (and they do — roughly 2,200 attacks happen every day), your credential gets added to "combo lists" and automatically tried on every major service. This is called credential stuffing and it's how most account takeovers happen.

The solution: Use a password manager (Bitwarden, 1Password, or KeePass) to store a unique 20+ character random password for every site. You only need to remember one master password.

The Modern NIST Guidelines (2024)

NIST SP 800-63B (updated 2024) completely reversed the old advice. Here's what they now recommend for both users and service providers:

  • Minimum 8 characters — but aim for 15+ for personal accounts
  • No mandatory complexity rules (no "must include a symbol") — length matters more
  • No mandatory periodic rotation unless there's evidence of compromise
  • Check against known breached password lists (Have I Been Pwned)
  • Enable multi-factor authentication whenever possible

Passphrases vs. Random Strings

If you need something memorizable, a passphrase (4–6 random words) is significantly better than a "complex" 8-character password. correct-horse-battery-staple has ~44 bits of entropy at 4 words from a 7,776-word Diceware list — far more than P@ssw0rd!.

However, for passwords you don't need to remember (which should be most of them), a fully random 20-character string is still stronger. Zecure's "Random Password" generator produces exactly this: true CSPRNG output, no dictionary words, no patterns.

What About Passphrases in 2025?

Passphrases are better for master passwords — the one password you actually type regularly. Use 5 truly random words (not your favorite quotes or song lyrics) from a large wordlist. For everything else, let a password manager generate 20+ random characters and store them.

Your Action Plan

  1. Install a password manager today — Bitwarden is free and open source
  2. Generate a strong master password — use 5 random Diceware words
  3. Enable 2FA on critical accounts — email, banking, work SSO
  4. Audit your existing passwords — most password managers flag reused/weak ones
  5. Replace weak passwords first — start with banking, email, and work accounts
