WPA2, WEP, and Why Your WiFi Password Matters

May 2025 7 min read By Zecure Team

WEP was cracked in 2007 and takes seconds to break today. WPA2 is only as secure as the password you choose — and most people choose badly. Here's what actually happens when you set your WiFi password, and why length matters more than complexity.

A Brief History of WiFi Security

When WiFi launched commercially in the late 1990s, the security protocol was WEP (Wired Equivalent Privacy). The name was optimistic. WEP used RC4 stream cipher with a static key and a critically flawed initialization vector (IV) scheme — IVs were only 24 bits and frequently reused. By 2001, researchers had published techniques to crack WEP keys by passively capturing enough packets.

Today, freely available tools like aircrack-ng can recover a WEP key in under a minute from a busy network, or in a few hours on a quiet one. WEP is not encryption — it's a feeling of security.

The WiFi Alliance responded with WPA (2003) and then WPA2 (2004), which replaced RC4 with AES-CCMP. WPA3 arrived in 2018 with stronger handshake protocols and forward secrecy. Most consumer routers since 2020 support WPA3, though many still default to WPA2 for compatibility.

How WPA2 Handshake Authentication Works

When a device connects to a WPA2 network, it performs a 4-way handshake with the router. The handshake derives a session key from three inputs: the network name (SSID), the password, and two random numbers exchanged during the handshake.

Critically, the handshake is transmitted over the air — and any attacker within range can capture it passively. They don't need to be connected to your network. Once captured, they take the handshake offline and try to crack it:

  1. Pick a candidate password from a wordlist or generate one
  2. Derive what the session key would be for that password + your SSID
  3. Check if that key matches the captured handshake
  4. Repeat — billions of times per second on a GPU

This is why your WiFi password matters so much: the attacker doesn't need to interact with your router at all after capturing the handshake. The only thing stopping them is the search space of your password.

PMKID attack (2018): Researchers discovered a technique that doesn't even require capturing a full handshake — attackers can request a single packet from your router and crack offline. WPA3 is immune to this; WPA2 is not.

WPA2 Password Requirements and Best Practices

WPA2-Personal accepts passwords from 8 to 63 characters. The WPA2 spec runs your password through PBKDF2 with 4096 iterations of HMAC-SHA1 to derive the Pre-Shared Key (PSK). Each additional character exponentially increases the cracking difficulty.

What makes a good WiFi password?

  • Length first — a 20-character lowercase phrase beats a 10-character "complex" one. Go for at least 20 characters.
  • Avoid real words combined with predictable substitutions — P@ssw0rd! is in every wordlist with rules applied
  • Don't use your SSID or address — attackers pre-compute rainbow tables for common SSIDs
  • Random is best — a 63-character random password using the full WPA2 character set is computationally uncrackable

WEP-128 and WEP-256: Already Broken

WEP keys are a different format — they're hex strings representing the raw key material, not passwords hashed into a key. WEP-128 uses a 104-bit key (26 hex chars) and WEP-256 uses a 232-bit key (58 hex chars).

The key length is irrelevant. WEP's vulnerability is in the RC4 IV reuse, not the key length. A longer WEP key doesn't meaningfully improve security because the attack works by statistical analysis of IVs, not by brute-forcing the key. WEP-256 can be cracked just as fast as WEP-128 on a busy network.

If your router only supports WEP, replace it. There is no configuration or password change that makes WEP secure. It is fundamentally broken at the protocol level.

WPA3: What's Different

WPA3-Personal replaces the WPA2 handshake with SAE (Simultaneous Authentication of Equals), also called Dragonfly. The key differences:

  • Forward secrecy — capturing traffic and cracking the password later doesn't decrypt previously captured sessions
  • Offline attack resistance — SAE doesn't expose a hash that can be cracked offline; each guess requires interacting with the access point (enabling rate limiting)
  • Dictionary attack protection — attackers can't grab a handshake and enumerate passwords at GPU speeds

WPA3 makes a weak password significantly more resilient — but a strong random password still makes both WPA2 and WPA3 uncrackable in practice.

Summary: What to Do Right Now

  1. If your router uses WEP — disable it and switch to WPA2 or WPA3 immediately
  2. Enable WPA3 if your router and devices support it; use WPA2/WPA3 transition mode if needed
  3. Set a random, 20+ character WiFi password (63 characters for maximum security)
  4. Change the default SSID — common names like "NETGEAR" have precomputed attack tables
  5. Disable WPS (WiFi Protected Setup) — it has its own crackable 8-digit PIN vulnerability
  6. Store your WiFi password in a password manager rather than writing it on a sticky note
