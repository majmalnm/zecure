Base64 Encoding Explained: What It Is and When to Use It

April 2026 7 min read By Zecure Team

You've seen Base64 strings everywhere — in JWTs, HTTP headers, data URIs, and email attachments. But what exactly is Base64, how does it work, and when should you actually use it? This guide answers all three questions.

What Is Base64?

Base64 is a binary-to-text encoding scheme that converts arbitrary binary data into a string of printable ASCII characters. The name "Base64" comes from the fact that it uses an alphabet of exactly 64 characters.

The standard Base64 alphabet consists of:

  • Uppercase letters: A–Z (26 characters)
  • Lowercase letters: a–z (26 characters)
  • Digits: 0–9 (10 characters)
  • Two special characters: + and / (2 characters)
  • Padding character: = (not part of the 64, used for alignment)

How Does Base64 Work?

Base64 works by grouping every 3 bytes (24 bits) of input data and encoding them as 4 Base64 characters (each representing 6 bits). Let's walk through an example with the word "Man":

ASCII
M (77) a (97) n (110)
Binary
01001101 01100001 01101110
6-bit groups
010011 010110 000101 101110
Base64
T (19) W (22) F (5) u (46)

So "Man" encodes to TWFu. This 3:4 byte-to-character ratio is why Base64 output is always about 33% larger than the original input.

What about padding?

Because Base64 works in groups of 3 bytes, if the input isn't a multiple of 3 bytes, padding is added:

  • 1 leftover byte → 2 Base64 characters + == padding
  • 2 leftover bytes → 3 Base64 characters + = padding
  • 0 leftover bytes → no padding needed

Standard Base64 vs URL-safe Base64 (Base64url)

The standard Base64 alphabet includes + and /. These characters have special meaning in URLs:

  • + in a URL query string means a space character
  • / is a path separator in URLs

If you embed a standard Base64 string in a URL without percent-encoding it, these characters break the URL. That's why Base64url (RFC 4648 §5) was created:

VariantCharacter 62Character 63PaddingUse case
Standard Base64+/= requiredEmail (MIME), general encoding
URL-safe (Base64url)-_= optionalJWT, OAuth, URL params
 
// JavaScript: Standard and URL-safe Base64
// Encode to standard Base64
const standard = btoa('Hello, World!');
// → "SGVsbG8sIFdvcmxkIQ=="

// Encode to URL-safe Base64
const urlSafe = btoa('Hello, World!')
  .replace(/\+/g, '-')
  .replace(/\//g, '_')
  .replace(/=+$/, '');
// → "SGVsbG8sIFdvcmxkIQ"

When Do Developers Use Base64?

1. HTTP Basic Authentication

When you send credentials with HTTP Basic Auth, the browser encodes username:password in Base64 and puts it in the Authorization header:

Authorization: Basic dXNlcjpwYXNzd29yZA==
// "dXNlcjpwYXNzd29yZA==" is Base64("user:password")

HTTP Basic Auth Base64 is not encrypted. Anyone who intercepts the request can decode it instantly. Always use HTTPS with Basic Auth.

2. JSON Web Tokens (JWT)

A JWT consists of three Base64url-encoded parts separated by dots: header.payload.signature. The header and payload are JSON objects encoded as Base64url. This makes them readable by anyone — JWTs are not encrypted by default, just signed.

3. Data URIs

To embed binary files like images or fonts directly in CSS/HTML without a separate HTTP request, use a data URI:

<!-- Embed a tiny PNG directly in HTML -->
<img src="data:image/png;base64,iVBORw0KGgoAAAANS..." />

/* Embed a font in CSS */
@font-face {
  src: url('data:font/woff2;base64,d09GRgAB...');
}

4. Email Attachments (MIME)

Email protocols (SMTP) were designed to carry only 7-bit ASCII text. Binary attachments like PDFs and images are Base64-encoded before embedding in the email body using MIME (Multipurpose Internet Mail Extensions).

5. Storing Binary Data in JSON or Databases

JSON has no binary type. When you need to store or transmit binary data (encryption keys, signatures, certificates, images) in a JSON field, Base64-encode it first.

What Base64 Is NOT

Base64 is not encryption and not security. It is completely reversible without a key. Never use Base64 to "protect" passwords, tokens, or sensitive data. For confidentiality, use proper encryption (AES-256-GCM). For authentication, use HMAC or digital signatures.

Quick Reference: Base64 in Popular Languages

// JavaScript (browser & Node.js)
const encoded = btoa('hello world');       // "aGVsbG8gd29ybGQ="
const decoded = atob('aGVsbG8gd29ybGQ=');  // "hello world"

// Python
import base64
encoded = base64.b64encode(b'hello world').decode()
decoded = base64.b64decode('aGVsbG8gd29ybGQ=').decode()

// Go
import "encoding/base64"
encoded := base64.StdEncoding.EncodeToString([]byte("hello world"))

// Terminal
echo -n 'hello world' | base64
echo 'aGVsbG8gd29ybGQ=' | base64 --decode
