What Makes a Strong Password?

A strong password depends on two things: length and character diversity. Every character you add makes brute-force attacks exponentially harder — a password with 20 characters using only lowercase letters has 2620 possible combinations, compared to 6210 for a 10-character alphanumeric password.

For accounts and services you access online, the minimum recommended is 12 characters using all four character classes. For high-security use cases — master passwords, disk encryption — aim for 20+ characters. The strongest passwords this generator can produce reach ~398 bits of entropy, which is computationally infeasible to crack by any method known today.

All passwords are generated locally in your browser. Zero data is ever sent to a server. Open DevTools → Network while generating to verify — you will see no outbound requests.

How to Use This Generator

  • Adjust the length — Drag the slider to set password length from 8 to 64 characters. Longer is always stronger.
  • Choose character types — Toggle uppercase (A-Z), lowercase (a-z), numbers (0-9), and symbols (!@#$%^&*). Using all four gives the largest character pool.
  • Copy and use — Click any password to copy it to your clipboard instantly.
  • Regenerate — Click Regenerate for a fresh set of passwords using the same settings.

Even the strongest password is useless if reused across sites. Use a password manager to store unique passwords for every account.
