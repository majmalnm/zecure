What Is a Passphrase?

A passphrase is a password built from multiple random words rather than a string of random characters. The XKCD comic #936 ("correct horse battery staple") demonstrated that four random common words are both easier to remember and harder to crack than a short random character password like Tr0ub4dor&3.

This generator uses the format adjective-noun-noun-####! — two content words, a 4-digit number, and a symbol. Each component is picked by window.crypto.getRandomValues(), the same CSPRNG used in TLS and operating system key generation. No Math.random(), no timestamp seeds.

Generated passphrases are never sent to a server. Open DevTools → Network and generate — you'll see zero outbound requests.

Passphrase vs. Random Password

Neither is universally better — they serve different needs. Here's how they compare:

Passphrase — Good for

  • Master passwords you must memorise
  • Typing on mobile or a TV remote
  • Sharing verbally without spelling out
  • WiFi passwords guests need to type
  • Disk encryption / full-disk passwords

Random key — Better for

  • API keys stored in env variables
  • JWT signing secrets
  • AES encryption keys
  • Any secret never typed by a human
  • Machine-to-machine authentication

Entropy Breakdown

Entropy measures how many random bits went into a password — higher bits means more guessing work for an attacker. Here's how each part of this passphrase contributes:

Adjective (54 words)54~5.75 bits
Noun #1 (66 words)66~6.04 bits
Noun #2 (66 words)66~6.04 bits
4-digit number (1000–9999)9000~13.1 bits
Symbol (7 options)7~2.81 bits
Total—~33.7 bits

33–34 bits resists online brute-force attacks (rate-limited to thousands of guesses per second) but is not sufficient for offline attacks against a compromised database. For higher-security use, add more words or use the AES-256 key generator (256 bits).

How to Use a Passphrase

  • Password manager master password — The most common use case. You need one passphrase you can type from memory; make it your vault key and let the manager handle the rest.
  • Full-disk encryption — macOS FileVault, Windows BitLocker, and LUKS on Linux all accept passphrases at boot time — before any password manager is available.
  • SSH key passphrase — Protects your private key file. You type it once per session (or use ssh-agent), so memorability matters.
  • WiFi WPA2-Personal — Must be typed by guests on their own devices. A passphrase is far friendlier than a random character string.

For everything else — accounts, APIs, tokens — use a password manager with a random 20-character password. Save passphrases for the handful of secrets you must remember.
