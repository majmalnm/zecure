What Is a JWT Secret?

A JWT (JSON Web Token) secret is the key used to create a cryptographic signature that proves a token hasn't been tampered with. When your server issues a JWT, it signs the header and payload using HMAC-SHA256 (HS256) or a stronger variant with this secret. When it receives a token, it re-computes the signature and checks that it matches — if not, the token is rejected.

The security of every JWT your application issues depends entirely on keeping this secret safe. A leaked or weak secret lets anyone forge valid tokens and impersonate any user in your system.

Never use a weak or human-readable JWT secret. Strings like "secret", "mysecretkey", or any word/phrase are trivially guessable and have been responsible for real-world token forgery attacks.

How to Use This Secret

1
Copy a secret above and store it as an environment variable: JWT_SECRET=your-64-char-key. Use a different secret per environment (dev, staging, prod).
2
Sign tokens using a JWT library with your secret:
// Node.js with jsonwebtoken
import jwt from 'jsonwebtoken';

const token = jwt.sign(
  { userId: user.id, role: user.role },
  process.env.JWT_SECRET,
  { expiresIn: '15m', algorithm: 'HS256' }
);

// Verify (throws if invalid)
const payload = jwt.verify(token, process.env.JWT_SECRET);
3
Never commit this secret to version control. Add .env to .gitignore and verify with git check-ignore -v .env before your first push.
4
Rotate on compromise — if a secret leaks, generate a new one immediately. All existing tokens become invalid (users will need to log in again).
