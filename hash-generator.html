Which Hash Algorithm Should You Use?

Not all hash functions serve the same purpose. Here's when to use each:

AlgorithmOutputSpeedSecurityUse Case
MD5128-bit / 32 hexVery fastBrokenFile checksums, non-security uses only
SHA-1160-bit / 40 hexFastBrokenLegacy systems — avoid for new work
SHA-256256-bit / 64 hexFastSecureFile integrity, HMAC, digital signatures
SHA-512512-bit / 128 hexFastSecureHigh-security contexts, 64-bit systems

Never hash passwords with these functions. SHA-256 and SHA-512 are designed to be fast — an attacker can test billions of guesses per second. Use bcrypt, Argon2id, or scrypt for password storage instead.

Common Uses for Cryptographic Hashes

  • File integrity verification — Download a file, compute its SHA-256 hash, and compare it against the publisher's published checksum. If they match, the file hasn't been tampered with.
  • API key storage — Store a SHA-256 hash of an API key in your database instead of the key itself. On each request, hash the incoming key and compare. Even a full database dump won't expose usable keys.
  • Data deduplication — Hash file contents to detect duplicates without comparing them byte-by-byte.
  • HMAC signing — Use SHA-256 inside HMAC (Hash-based Message Authentication Code) to prove a message hasn't been modified and came from a trusted source.
  • Content-addressable storage — Store and look up data by its hash (e.g., Git uses SHA-1 / SHA-256 for commit and blob identifiers).

Frequently Asked Questions

A hash function takes any input and produces a fixed-length output called a digest or hash. The same input always produces the same hash, but even a tiny change in input produces a completely different hash. Critically, it's a one-way function — you cannot reverse a hash to recover the original input.

MD5 is vulnerable to collision attacks — two different inputs can produce the same MD5 hash. In 2004, researchers demonstrated practical collisions. In 2008, a rogue CA certificate was forged using MD5 collisions. For any security purpose (signatures, certificates, password storage), MD5 must not be used.
