The Error Code Ranges

HTTP defines codes from 100–599. We have extended this with three additional ranges covering error conditions the specification failed to anticipate:

  • 2xx Success — Things worked. We're all surprised. 200 OK carries more existential weight than it lets on.
  • 4xx Client Errors — You did something wrong. We're pretty sure. Most error pages live here.
  • 5xx Server Errors — We did something wrong. Possibly the entire team. 500 is the error that has ended careers.
  • 6xx Invented — General — Covers situations the HTTP spec naively assumed wouldn't happen. It happened.
  • 7xx Invented — Corporate — For when the error is organisational rather than technical. Regrettably common.
  • 9xx Invented — User Error — PEBCAK range. These are the caller's fault. We're being honest about it for once.

Technically, HTTP clients will treat anything in the 6xx range as an unknown server error in the 5xx family per spec. Practically, they'll break in interesting ways.
