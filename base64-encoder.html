What Is Base64?

Base64 is a binary-to-text encoding scheme that represents binary data as a sequence of printable ASCII characters. It maps every 3 bytes of input to 4 characters from a 64-character alphabet: A–Z, a–z, 0–9, +, and /. Padding characters (=) are added at the end if the input isn't a multiple of 3 bytes.

The result is ~33% larger than the input but can be transmitted safely over any channel that handles ASCII text — including email bodies, JSON payloads, HTTP headers, and HTML attributes.

When Do Developers Use Base64?

  • HTTP Basic Auth — Credentials are Base64-encoded as username:password before being placed in the Authorization: Basic header.
  • JSON Web Tokens (JWT) — JWT headers and payloads are URL-safe Base64-encoded (Base64url). This makes them safe in URL query strings without percent-encoding.
  • Embedding images or fonts in CSS/HTML — Data URIs use Base64 to inline binary files: src="data:image/png;base64,..."
  • Email attachments (MIME) — Binary attachments like images or PDFs are Base64-encoded before embedding in email bodies.
  • Storing binary data in JSON — JSON has no binary type, so binary blobs (keys, signatures, certificates) are Base64-encoded first.

Base64 is not encryption. Anyone can decode a Base64 string instantly — there is no key or secret involved. Never rely on Base64 to protect sensitive data. Use proper encryption (AES-256-GCM) for confidential information.

Standard vs URL-safe Base64

Standard Base64 uses + and / as the 62nd and 63rd characters. These are special characters in URLs — + means a space, and / is a path separator — which causes problems when Base64 strings appear in URLs.

URL-safe Base64 (also called Base64url) solves this by replacing + with - and / with _. The padding = is also often omitted. This variant is used in JWTs, Google's APIs, and most modern web authentication protocols.

// JavaScript: encode to URL-safe Base64
function toBase64Url(str) {
  return btoa(str)
    .replace(/\+/g, '-')
    .replace(/\//g, '_')
    .replace(/=+$/, '');
}
