What Is an AES-256 Key?

An AES-256 key is 32 bytes (256 bits) of cryptographically random data used with the AES symmetric cipher. Represented as hexadecimal, it becomes 64 characters (2 hex digits = 1 byte). This is the key size approved by the NSA for Top Secret classified information and used by AWS, Apple, Signal, and every HTTPS connection.

The key must be generated by a CSPRNG — this tool uses crypto.getRandomValues(new Uint8Array(32)) in your browser. A key derived from a password, timestamp, or any predictable source is cryptographically weak regardless of how many bits it is.

AES-256 has never been broken in practice. The only vulnerabilities are in the key — weak generation, hardcoded values, or inadequate storage. The algorithm itself is sound.

How to Use This Key in Node.js

// Store key as hex in AES_KEY env variable, then:
import crypto from 'crypto';

function encrypt(plaintext) {
  const key = Buffer.from(process.env.AES_KEY, 'hex'); // 32 bytes
  const iv  = crypto.randomBytes(12);              // 96-bit IV for GCM
  const cipher = crypto.createCipheriv('aes-256-gcm', key, iv);
  const enc = Buffer.concat([cipher.update(plaintext), cipher.final()]);
  return { iv: iv.toString('hex'), data: enc.toString('hex'), tag: cipher.getAuthTag().toString('hex') };
}
  • Always use AES-GCM mode — it provides both encryption and authentication
  • Generate a fresh random IV for every encryption operation (never reuse an IV)
  • Store the IV and auth tag alongside the ciphertext — they are not secret
  • The key itself must be kept secret and rotated when compromised

Frequently Asked Questions

An AES-256 key is 32 bytes (256 bits) of random data used to encrypt and decrypt data with the AES cipher. It's written as 64 hexadecimal characters. The same key is used for both encryption and decryption (symmetric encryption).

Use AES-GCM for new applications — it's an authenticated encryption mode that provides both confidentiality and integrity verification. Avoid ECB mode (patterns in plaintext leak into ciphertext). AES-CBC is acceptable if combined with HMAC-SHA256, but GCM is simpler to use correctly.
